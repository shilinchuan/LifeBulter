# LifeButler 演示视频指导手册

适用场景：课程最终提交演示视频

建议时长：6 到 8 分钟

建议录制比例：16:9，分辨率 1920×1080 或 2560×1440

项目链接：https://github.com/shilinchuan/LifeBulter

Release 链接：https://github.com/shilinchuan/LifeBulter/releases/tag/v0.3.0-final

## 一、录制前准备

### 1. 环境准备

建议使用已经打包好的 macOS App 录制，减少终端环境干扰：

```text
dist/LifeButler.app
```

如果 App 无法直接打开，可以右键点击 App，选择“打开”。因为该包是本机 ad-hoc 签名，不是 Apple 官方公证包，首次打开可能需要手动允许。

### 2. 数据准备

录制前建议准备少量演示数据，避免空页面影响观感：

1. 至少 2 条记账记录：一条支出、一条收入。
2. 至少 1 个 Objective、1 个 KR、1 个 Project。
3. 至少 4 个任务，分别加入今日四象限。
4. 至少 1 条体重记录、1 条运动记录。
5. 至少 1 条备忘录。
6. 至少 1 条周计划任务和一段周报复盘。

不要展示个人隐私数据。若使用真实数据库，录制前先备份。

### 3. 录制建议

1. 鼠标移动慢一些，点击后停留 1 到 2 秒。
2. 每个页面只讲一个核心点，不要在同一页解释太多细节。
3. 演示过程中尽量使用深色主题，最后切换一次浅色主题证明主题支持。
4. 如果出现弹窗，先说明弹窗作用，再点击关闭或保存。
5. 视频结尾必须展示 GitHub 仓库和 Release 链接。

## 三、逐段演示脚本

### 片段 1：开场与项目定位

画面操作：

1. 打开 GitHub 仓库页面。
2. 简单展示仓库名、README 和 Release。
3. 切到 LifeButler App 主界面。

建议讲解词：

> 大家好，我们的课程项目是 LifeButler 智能生活管家。这是一个本地运行的桌面生活管理应用，技术栈是 PyQt6、SQLite 和 matplotlib。它的核心目标是把日常生活中的目标、任务、记账、健康记录、备忘录和周报复盘整合到一个系统里，形成“计划、执行、记录、反馈”的生活管理闭环。项目源码已经上传到 GitHub，最终 macOS App 也放在 GitHub Release 中。

停留重点：

- GitHub 仓库是公开的。
- Release 中有可下载的 macOS zip。
- 项目是本地应用，不是网页应用。

### 片段 2：首页驾驶舱与快速收集

画面操作：

1. 打开首页驾驶舱。
2. 指向今日三件事、今日专注、本月财务、近 7 天运动和生活雷达。
3. 在快速收集框输入一个例子，例如 `午饭 28r`。
4. 展示系统识别为账单确认弹窗。
5. 再输入 `跑步 30 分钟` 或 `明天提交报告`，展示识别能力。

建议讲解词：

> 首页驾驶舱是用户打开应用后的总览页面。这里把今日任务、专注情况、本月财务、近期运动和生活雷达聚合在一起，用户不需要进入每个模块就能判断当天状态。上方的快速收集是降低记录成本的入口，例如输入“午饭 28r”，系统会用本地规则识别为记账；输入“跑步 30 分钟”会识别为运动记录；输入普通事项则会进入任务或备忘录流程。这里没有接入外部 AI API，规则是本地可解释的。

停留重点：

- 快速收集支持 `r/R` 作为“元”的别名。
- 保存前有确认弹窗。
- 首页不是装饰页，而是实际工作台。

### 片段 3：记账管理

画面操作：

1. 切到“记账管理”页面。
2. 展示收支表格。
3. 点击新增记录，展示字段：类型、分类、金额、日期、备注。
4. 展示月度收入、支出、结余。
5. 展示支出 / 收入分类图表切换。

建议讲解词：

> 记账模块负责日常收支记录。用户可以新增、编辑和删除账单，系统会按月统计收入、支出和结余。右侧图表使用 matplotlib 绘制，并通过统一的 ChartWidget 适配深浅主题。这个模块的目标是轻量记录和快速反馈，不做复杂财务系统。

停留重点：

- 录入和统计形成闭环。
- 图表能展示分类占比。
- 数据保存到本地 SQLite。

### 片段 4：目标管理

画面操作：

1. 切到“目标管理”。
2. 展示 Objective、KR、Project 三栏。
3. 双击 Objective 行，打开详情窗口。
4. 双击 KR 行，展示 KR 当前值 / 目标值和更新进度入口。
5. 双击 Project 行，展示项目详情。

建议讲解词：

> 目标管理模块把生活管理从单个任务扩展到目标推进。这里采用 Objective、KR、Project 三层结构。Objective 表示阶段目标，KR 表示可量化的关键结果，Project 表示目标下的推进对象。根据实际使用反馈，我们移除了任务和项目之间的绑定关系，避免用户误以为项目是任务容器。现在 Project 只属于 OKR 体系，任务则属于待办、今日四象限和周计划。

停留重点：

- 顶部只保留新增入口。
- 编辑和删除在详情窗口中完成。
- KR 表格紧凑显示 `当前值/目标值 单位`。

### 片段 5：待办、今日四象限和番茄钟

画面操作：

1. 切到“待办事项”。
2. 展示任务池，新增或编辑一个任务。
3. 强调双击任务是编辑，不是切换完成。
4. 使用按钮把任务加入今日四象限。
5. 切到今日四象限页面。
6. 选择一个今日任务，点击番茄钟开始、暂停、结束。

建议讲解词：

> 待办模块包含任务池和今日四象限。任务池负责维护所有任务，今日四象限负责安排今天要执行的任务。四象限分为重要紧急、重要不紧急、不重要紧急和不重要不紧急。未完成的今日任务会自动顺延显示，已完成任务不会继续出现在今日视图。右侧番茄钟可以绑定今日任务，完成一次专注后会记录番茄 session，并统计任务当天的番茄数和专注分钟。

停留重点：

- 任务双击进入编辑。
- 完成 / 恢复通过按钮触发。
- 番茄钟和今日任务绑定。

### 片段 6：健康记录

画面操作：

1. 切到“健康记录”。
2. 展示体重记录和 BMI。
3. 展示体重趋势图。
4. 切到运动记录，展示运动时长和周统计图。

建议讲解词：

> 健康模块用于记录身体状态和运动行为。体重记录会自动计算 BMI，并展示趋势图。运动记录可以记录运动类型、时长和日期，并用图表展示本周运动分布。这样 LifeButler 不只是管理任务，也把生活状态纳入反馈范围。

停留重点：

- BMI 自动计算。
- 体重和运动都有图表。
- 与生活管理主题一致。

### 片段 7：备忘录

画面操作：

1. 切到“备忘录”。
2. 展示分类、搜索、置顶。
3. 打开或新增一条 Markdown 内容。
4. 展示预览效果。

建议讲解词：

> 备忘录模块用于保存非结构化信息，比如想法、摘录、临时记录和学习笔记。它支持分类、搜索、置顶和 Markdown 预览。这个模块的作用是承接那些不适合放进任务、目标或账单里的生活信息。

停留重点：

- 非结构化信息管理。
- Markdown 预览。
- 搜索和置顶提升可用性。

### 片段 8：周计划与周报复盘

画面操作：

1. 切到“周计划 / 周报”页面。
2. 展示本周任务列表。
3. 调整一个任务完成度或填写进展备注。
4. 标记为今日或取消今日。
5. 填写三句复盘。
6. 点击导出 Markdown。

建议讲解词：

> 周计划模块把任务从当天扩展到一周维度。用户可以把任务加入本周，记录完成度和进展备注，也可以把周任务标记为今日任务。周报复盘提供三句复盘，并支持导出 Markdown。这样系统不只是记录执行，还能把一周的结果沉淀为可保存的复盘材料。

停留重点：

- 周计划和今日任务互通。
- 支持进展备注。
- 支持 Markdown 导出。

### 片段 9：设置、主题和备份

画面操作：

1. 切到“设置”页面。
2. 展示数据库路径。
3. 点击或展示备份数据库按钮。
4. 切换深色 / 浅色主题。

建议讲解词：

> 设置页面展示本地数据库路径，并提供数据库备份入口。LifeButler 的数据默认保存在本机，不上传服务器。应用也支持深色和浅色主题切换，主题切换会同步影响 Qt 控件和 matplotlib 图表。为了保证打包版不会把数据库写进 App 内部，macOS App 默认会把数据放到用户的 Application Support 目录。

停留重点：

- 数据本地保存。
- 支持备份。
- 深浅主题联动。

### 片段 10：技术实现与收尾

画面操作：

1. 切回 GitHub README。
2. 展示项目目录结构。
3. 展示 tests 目录。
4. 展示 Release 页面和 App zip。

建议讲解词：

> 从技术实现上看，项目采用 PyQt6 桌面界面、SQLite 本地数据库和 matplotlib 图表。DatabaseManager 统一管理连接、数据 API 和 schema migration，当前数据库版本是 v5。服务层负责目标、周计划、快速收集和首页概览等逻辑，UI 层只负责展示和交互。我们也提供了 JSON CLI，方便脚本或 Agent 获取今日概览、任务、目标和周报信息。最终提交前，我们运行了 py_compile 和 unittest，23 个测试全部通过。源码已经上传到 GitHub，macOS App 放在 Release 附件中。

结尾建议讲解词：

> 以上就是 LifeButler 智能生活管家的最终演示。它已经完成从源码、测试、文档到 macOS App 打包和 GitHub Release 的完整提交流程。谢谢观看。

停留重点：

- GitHub 链接清楚可见。
- Release 附件清楚可见。
- 测试通过可以口头说明，不必现场运行测试。

## 四、推荐录屏操作清单

录屏时按下面顺序执行：

1. 打开 GitHub 仓库。
2. 打开 Release 页面。
3. 打开 LifeButler App。
4. 首页输入 `午饭 28r`，展示账单识别。
5. 首页输入 `跑步 30 分钟`，展示运动识别。
6. 打开记账页面，展示统计图。
7. 打开目标页面，双击 Objective / KR / Project 查看详情。
8. 打开待办页面，双击任务进入编辑。
9. 切到今日四象限，演示番茄钟开始、暂停、结束。
10. 打开健康页面，展示体重和运动图表。
11. 打开备忘录页面，展示 Markdown 预览。
12. 打开周报页面，展示完成度、复盘和导出。
13. 打开设置页面，展示数据库路径和备份。
14. 切换一次浅色主题。
15. 回到 GitHub Release 页面收尾。

## 五、常见失误与规避

1. 不要在视频里展示真实私人账单或数据库路径中的敏感信息。如果路径里有个人目录名，可以快速带过，不要放大。
2. 不要现场安装依赖。视频重点是成品演示，不是环境配置。
3. 不要在演示中长时间等待番茄钟。只展示开始、暂停和结束即可。
4. 不要解释每一个按钮。只讲核心流程和设计意义。
5. 如果 App 首次打开提示安全限制，提前右键打开处理好，不要录进正式视频。
6. 如果 GitHub 页面加载慢，可以提前打开仓库和 Release 标签页。
